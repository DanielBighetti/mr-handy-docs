Veja em ação

O que é fisicamente o Mr. Handy?

Mr. Handy é um workspace operacional que conecta IDE, Figma, agentes de IA, MCPs e documentação estruturada para transformar telas e componentes em handoffs prontos para execução.

Outputs principais
01 · Mode A

Markdown, HTML navegável e screenshots por tela para handoff fora do Figma.

02 · Mode B

Documentação criada diretamente no canvas do Figma, com SpecCards e notas visuais.

03 · Component Spec

Especificação profunda de componentes, variantes, tokens, estrutura, motion e acessibilidade.

04 · Run Log

Rastreabilidade da execução, cobertura, fases, reroutes e validações do pipeline.

Instalação

Ambiente necessário

O Mr. Handy opera com Node.js, Cursor ou IDE compatível, Figma OAuth e plugin uSpec Extract para o fluxo de componentes.

Node.js 18+Runtime necessário para setup e ferramentas.
Cursor / IDEAmbiente onde os comandos e MCPs são carregados.
Figma OAuthAutorização via navegador, sem PAT manual.
uSpec ExtractPlugin usado no caminho de especificação de componentes.

FAQ

Perguntas frequentes

O Mr. Handy cria design?

Não. Ele lê designs prontos e gera documentação estruturada para execução, validação e handoff.

Precisa de token manual do Figma?

Não. O fluxo atual usa OAuth via navegador.

Essa versão substitui a documentação atual?

Não. Esta é uma versão v2 independente. A versão anterior permanece publicada.
